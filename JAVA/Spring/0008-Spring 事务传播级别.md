# Spring事务传播级别

| 传播行为          | 核心规则             | 典型应用场景                 |
|---------------|------------------|------------------------|
| REQUIRED (默认) | 有事务就加入，没事务就新建    | 绝大多数业务增删改操作（默认值）       |
| REQUIRES_NEW  | 挂起当前事务，新建一个独立事务  | 记录日志、发送通知（无论主业务成败都要提交） |
| NESTED        | 嵌套事务（基于保存点）      | 主流程失败子流程回滚，子流程失败可独立回滚  |
| SUPPORTS      | 有事务就加入，没事务就非事务执行 | 查询操作（兼容事务和非事务环境）       |
| MANDATORY     | 必须有事务，否则抛异常      | 强制要求在事务上下文中调用的核心方法     |
| NOT_SUPPORTED | 挂起当前事务，以非事务执行    | 复杂的查询或耗时操作，不想占用事务资源    |
| NEVER         | 必须非事务，否则抛异常      | 绝对不能在事务中执行的代码（极少用）     |

不正确的嵌套级别会发生错误
