# 缓存击穿/缓存穿透/缓存雪崩

| 问题   | 核心原因           | 形象比喻                      | 解决方案              |
|------|----------------|---------------------------|-------------------|
| 缓存穿透 | 查询根本不存在的数据     | 黑客拿着假钥匙疯狂捅锁眼，试图打开不存在的门。   | 布隆过滤器、缓存空值        |
| 缓存击穿 | 热点数据过期，高并发瞬间涌入 | 一个热门景点的检票口突然坏了，所有人瞬间挤进大门。 | 互斥锁、热点数据永不过期      |
| 缓存雪崩 | 大量数据同时过期，或缓存宕机 | 整个停车场的栏杆同时抬起，所有车瞬间涌向出口。   | 随机过期时间、多级缓存、高可用架构 |

详细解析

## 缓存击穿

* 这是什么： 指某个热点 Key（比如微博热搜第一名、秒杀商品）在缓存中过期的那一瞬间，恰好有大量并发请求过来。因为缓存里没有数据了，这些请求会直接穿透到数据库，导致数据库压力瞬间飙升。
* 关键点： 单个热点 Key、高并发、过期瞬间。
* 怎么防：  
  互斥锁： 只让一个线程去查数据库重建缓存，其他线程等待。  
  逻辑过期： 设置缓存永不过期，在值里面包含一个逻辑过期时间，后台异步去刷新。

## 缓存穿透

* 这是什么： 指请求查询的数据在缓存和数据库中都不存在（比如查询 ID 为 -1 或 999999
  的数据）。因为缓存里没有，数据库里也没有（无法回写缓存），导致每次请求都会直接打到数据库，就像穿透了缓存层一样。
* 关键点： 数据不存在、恶意攻击或逻辑错误。
* 怎么防：
  [[0001-布隆过滤器]]： 在请求到达缓存前先判断 Key 是否合法，不合法直接拦截。  
  缓存空值： 即使数据库查不到，也在缓存里存一个 null 值，并设置一个较短的过期时间（如 5 分钟）。

## 缓存雪崩

* 这是什么： 指在某一时刻，大量的 Key 同时过期，或者缓存服务宕机（如 Redis 挂了）。这会导致所有请求瞬间全部涌向数据库，数据库可能因此崩溃，引发整个系统的连锁反应。
* 关键点： 大规模 Key 同时失效、缓存服务不可用。
* 怎么防：  
  随机过期时间： 给过期时间加一个随机值（例如 1 小时 + 随机 1-10 分钟），避免同一时刻集体失效。  
  高可用架构： 搭建 Redis 集群（主从+哨兵），防止单点故障。

## 一句话总结：

* 穿透是查“没有的东西”；
* 击穿是“热点”突然没了；
* 雪崩是“一大片”都没了。
